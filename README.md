# Nebula Vanguard — Space Invaders

A story-driven, mobile-friendly, **offline-first** Space Invaders built with **TypeScript + HTML5 Canvas + Vite**, with a **global leaderboard backed by Supabase** (free tier).

## Features

- 🎮 Classic Space Invaders mechanics: swarms, shields, increasing waves.
- 📜 **Story-based & interactive** — intro narrative plus dialogue beats between levels (all data-driven in `src/game/levels.ts`).
- 🌌 **Polished graphics** — parallax starfield, neon/glow rendering, particle explosions, screen shake (all drawn procedurally, no image assets to download).
- 📱 **Mobile + desktop** — on-screen touch buttons, drag-to-move, and keyboard controls; responsive canvas that scales to any viewport.
- 📶 **Low-internet friendly** — tiny asset-light bundle, synth audio (no audio files), and a **PWA service worker** so it loads and plays fully offline after the first visit.
- 🏆 **Global leaderboard** via Supabase, with **offline queueing** that syncs automatically when the connection returns.
- 👾 **Intentionally unbeatable final boss** — "The Hive Mind" has a massive HP pool and a multi-phase attack pattern that *ramps up as its health drops*. It's beatable in theory, effectively impossible in practice (by design — see the note in `src/game/levels.ts`).

## Run locally

```bash
npm install
cp .env.example .env   # fill in your Supabase values (optional for local-only play)
npm run dev
```

Open the printed URL. Without Supabase credentials the game runs in **local-only leaderboard mode** (scores saved in the browser).

## Build & preview

```bash
npm run build     # type-checks then builds to dist/
npm run preview   # serves the production build locally
```

The `dist/` output is a static site — deploy it to GitLab Pages, Netlify, Vercel, or any static host.

## Supabase setup (free tier)

1. Create a free project at <https://supabase.com>.
2. In **Project Settings → API**, copy the **Project URL** and the **anon public** key into your `.env`:
   ```env
   VITE_SUPABASE_URL=https://your-ref.supabase.co
   VITE_SUPABASE_ANON_KEY=your-anon-key
   ```
   The anon key is safe to ship in a client app **as long as Row Level Security is enabled** (below).
3. In the Supabase **SQL Editor**, run:

   ```sql
   create table if not exists public.leaderboard (
     id          bigint generated by default as identity primary key,
     player      text not null check (char_length(player) between 1 and 16),
     score       integer not null check (score >= 0),
     level       integer not null default 1,
     created_at  timestamptz not null default now()
   );

   create index if not exists leaderboard_score_idx on public.leaderboard (score desc);

   -- Enable Row Level Security so the public anon key is safe.
   alter table public.leaderboard enable row level security;

   -- Anyone may read the leaderboard.
   create policy "public read"
     on public.leaderboard for select
     to anon
     using (true);

   -- Anyone may submit a score (insert only; no update/delete).
   create policy "public insert"
     on public.leaderboard for insert
     to anon
     with check (true);
   ```

4. Rebuild/restart so Vite picks up the env vars. Scores now sync to the global board; offline scores queue locally and upload when you're back online.

> Tip: to harden against spam you can add rate limiting or a Postgres trigger, but the policies above are enough to get a working public leaderboard.

## Project structure

```
src/
  core/        engine primitives: types/math, input, audio, leaderboard (Supabase + offline sync)
  game/
    entities.ts  player, enemies, boss, bullets, shields
    fx.ts        starfield + particle systems
    levels.ts    data-driven levels + story beats (edit the campaign here)
    ui.ts        DOM overlay screens (menu, story, leaderboard, game over)
    game.ts      scene state machine + main loop
  main.ts      entry point
```

## Tuning the campaign

Everything narrative and difficulty-related lives in `src/game/levels.ts`: add levels to `LEVELS`, edit story text, or adjust enemy counts/speed/fire rate. The final entry (`isBoss: true`) is the Hive Mind boss.
